# Clickhouse_Optimisation 🚀

Репозиторий содержит скрипты для генерации тестовых данных логов в сжатый файл формата CSV (GZIP) и выполнения SQL-запросов в СУБД ClickHouse.

## 📋 Структура проекта

* **`main.sql`** — файл со структурой таблицы, тестовым агрегирующим запросом и командами для удаления данных и проверки мутаций.
* **`main.py`** — скрипт на чистом Python (использует встроенные модули `csv`, `gzip`, `random`, `time`, `datetime`) для генерации 10 000 000 строк тестовых логов.

---

## 💾 Описание структуры данных

Таблица `logs_ch` хранит логи со следующими полями:
* `timestamp` (`DateTime`) — дата и время лога.
* `user_id` (`UInt64`) — идентификатор пользователя.
* `url` (`String`) — запрашиваемый URL.
* `response_time` (`UInt32`) — время ответа.
* `status_code` (`UInt16`) — HTTP-код ответа.

Индексация таблицы выполняется по ключу: `ORDER BY (user_id, timestamp)`.

---

## 🚀 Инструкция по использованию

### 1. Генерация тестовых данных
Скрипт `main.py` генерирует 10 000 000 строк батчами по 500 000 строк и сохраняет их в сжатый архив `logs_ch_data.csv.gz` с разделителем табуляции (`\t`). При генерации учитываются веса для HTTP-статусов (код 200 является преобладающим).

Для запуска скрипта не требуются сторонние библиотеки:
```bash
python main.py
```

### 2. Создание таблицы и работа с SQL
Последовательность выполнения команд в `main.sql`:

1. **Создание таблицы:**
   ```sql
   CREATE TABLE logs_ch (
        timestamp DateTime,
        user_id UInt64,
        url String,
        response_time UInt32,
        status_code UInt16
    ) ENGINE = MergeTree()
    ORDER BY (user_id, timestamp);
   ```

2. **Тестовая агрегация:**
   Выборка количества логов по пользователям за период после `2026-04-19`:
   ```sql
   SELECT 
       user_id, 
       count(*) 
   FROM logs_ch 
   WHERE timestamp > '2026-04-19'::date 
   GROUP BY user_id;
   ```

3. **Удаление записей и мониторинг:**
   Удаление данных пользователя и отслеживание статуса выполнения асинхронной мутации на сервере ClickHouse:
   ```sql
   DELETE FROM logs_ch WHERE user_id = 1;

   SELECT * FROM system.mutations;
   ```
